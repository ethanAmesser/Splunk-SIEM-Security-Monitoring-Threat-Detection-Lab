# 🔐 Splunk SIEM Security Monitoring & Threat Detection Lab

## 📌 Overview
This project demonstrates the use of Splunk SIEM for log analysis, security monitoring, and threat detection using the BOTS v3 dataset. The lab simulates real-world activity, allowing for hands-on experience in identifying suspicious behavior and creating detection mechanisms.

## 🛠️ Tools & Technologies
- Splunk Enterprise
- SPL (Search Processing Language)
- BOTS v3 Dataset
- Windows Event Logs

## 🎯 Objectives
- Analyze authentication logs for suspicious behavior
- Detect brute-force login attempts
- Identify abnormal file modification activity
- Create visualizations to monitor log activity
- Configure alerts for automated threat detection

## ⚙️ Project Activities
- Ingested and indexed log data into Splunk
- Queried and filtered logs using SPL
- Analyzed authentication events (EventCode 4624 & 4625)
- Built visualizations to identify trends and anomalies
- Created alerts for brute-force detection and suspicious activity

## 🚨 Key Findings
- Identified repeated failed login attempts indicating possible brute-force attacks
- Detected spikes in authentication activity across hosts
- Observed abnormal file modification behavior
